FIAwrapper Function Validation Against Real FIA Data
=====================================================
Date: 2026-03-14
Data Source: rFIA fiaRI dataset (Rhode Island, real FIA inventory data)
             10,644 trees, 702 plots, 908 conditions, 60 species
             Inventory years 2004-2018, 337 sampled plots
Reference: EVALIDator methodology (TPA_EXP = TPA_UNADJ * CONDPROP_UNADJ)

=== BUGS IDENTIFIED AND FIXED ===

Bug 1: CONDPROP_UNADJ applied backwards in compute_tpa()  [FIXED]
  File: R/expansion.R line 98
  Was: tpa / cond_prop (DIVIDES - inflates TPA for partial conditions)
  Fix: tpa * cond_prop (MULTIPLIES - per EVALIDator methodology)
  Evidence: test-evalidator-crosscheck.R line 23: TPA_EXP = TPA_UNADJ * CONDPROP_UNADJ
  Impact: On real RI data with mean CONDPROP=0.77, TPA inflated ~30-70%

Bug 2: n_subplots auto-detected from data instead of using FIA design constant  [FIXED]
  File: R/expansion.R line 85
  Was: n_subplots <- length(unique(subplot))
  Fix: Added n_subplots = 4L parameter (standard FIA 4-subplot design)
  Impact: Plots with trees on <4 subplots got inflated TPA

Bug 3: No COND_STATUS_CD or PLOT_STATUS_CD filtering  [FIXED]
  File: R/prep_data.R
  Was: No condition/plot status filters
  Fix: Filter COND to COND_STATUS_CD == 1, PLOT to PLOT_STATUS_CD == 1
  Impact: Non-forest conditions and non-sampled plots included

Bug 4: summarize_growth() produced Inf/NA for ingrowth-only groups  [FIXED]
  File: R/time_series.R
  Was: min(inventory_year - years_since_previous, na.rm=TRUE) -> Inf
  Fix: Filter out groups with all-NA years_since_previous, use median
  Impact: 6 Inf and 55 NA records eliminated

Bug 5: No design-based estimation using FIA POP tables  [FIXED]
  File: R/variance.R
  Was: Only SRS and post-stratified estimation; no POP table support
  Fix: Added estimate_population() function using ratio-of-means estimator
       with POP_STRATUM weights and ADJ_FACTOR adjustments
  Impact: estimate_variance() returned BA=80.5 (SRS mean)
          estimate_population() returns BA=109.2 (design-based, closer to ~122)
